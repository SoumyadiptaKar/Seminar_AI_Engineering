# Gradient Ascent (GA-style) Unlearning

## Base Architecture
- Backbone model: Ultralytics YOLO (`weights.pt` from `backend/models`).
- Strategy implemented here: **two-stage GA-style unlearning**.
  1. Forget-stage suppression on forget images with empty labels.
  2. Retain-stage recovery fine-tuning on retain dataset.

## Inputs
- `original_weights`: starting model checkpoint.
- `split_manifest`: generated by `experiments/split_dataset.py`.
- `dataset_root`: raw image root.
- Training config: `device`, `batch_size`, `learning_rate`, `imgsz`, `workers`, `seed`.

## Outputs
- `outputs/gradient_ascent/unlearned.pt`
- prepared intermediate datasets under `outputs/gradient_ascent/prepared_data/`
- training logs under `outputs/gradient_ascent/forget_stage*` and `retain_stage*`

## Main Hyperparameters
- `gradient_ascent.forget_epochs`
- `gradient_ascent.retain_epochs`
- `run.learning_rate`
- `run.batch_size`
- `run.imgsz`
- `run.device` (`auto` resolves to `mps` on Apple Silicon)
- `gradient_ascent.dry_run`

## Notes
- This implementation is practical for class forgetting experiments in this repository.
- It is closer to a suppression+recovery unlearning baseline than strict signed-loss GA on ground-truth detection loss.
